Semaine 2

Introduction

Cette semaine, nous souhaitons vous faire approfondir la compilation séparée (Makefile) et utiliser une seconde bibliothèque qui sera nécessaire plus tard à notre projet (libfuse). Les objectifs pédagogiques visés par cette semaine sont :

  1. que vous connaissiez les bases des Makefile : savoir lire un Makefile simple et savoir le compléter sur des cas simples ;

  2. être prêt(e)s pour l'utilisation de la bibliothèque libfuse lorsque ce sera nécessaire (semaine 8) ; s'assurer que tout l'environnement nécessaire est déjà bien en place.

Nous vous proposons pour cela trois étapes :

  1. comprendre (lecture) puis compléter un Makefile d'un projet très simple ;
  2. écrire le Makefile d'un projet plus ambitieux déjà fourni ;
  3. comprendre un exemple simple fourni utilisant libfuse.

Vous aurez à rendre le(s) code(s) correspondant(s), avec ceux de la semaine passée et de la semaine suivante, d'ici au dimanche 19 mars 23:59.

II. Savoir rentrer dans un plus gros projet

Une des compétences qui vous sera aussi utile à l'avenir (hors de ce cours) est de savoir appréhender de gros projets déjà existants. Dans cet esprit, et aussi afin de vous faire pratiquer encore un peu plus les Makefile, nous fournissons (dans provided/bigprj) un projet C déjà existant, dont vous avez à écrire le Makefile (à faire dans done/bigprj -- pour rappel, le répertoire provided NE doit PAS être modifié).

REMARQUE IMPORTANTE : le code fourni ici est soumis à droit d'auteur et ne doit en aucun cas être recopié ailleurs, ni réutilisé en aucune façon.
(Il est par ailleurs critiquable de plusieurs points de vue et n'est donc pas du tout fourni en tant que source d'inspiration...)
Il vous est fourni uniquement pour que vous écriviez un Makefile permettant de le compiler.

Le code fourni utilise des sous-répertoires et une fonction de C99 (roundf()).

Pour indiquer de rechercher des fichiers d'en-tête dans des sous-répertoires, il faut ajouter une option -I par sous-répertoire voulu. Par exemple, lors de la compilation de machin.c, pour demander au compilateur de rechercher un fichier d'en-tête dans le sous-répertoire truc, il faut faire :

gcc -c -I truc machin.c -o machin.o

(Voir aussi la variable CFLAGS dans la documentation sur make fournie la semaine passée.)

Pour compiler suivant la norme C99 (ou au dessus), passer l'option -std=c99 (ou -std=c17 ou -std=c2x) au compilateur (voir la variable CFLAGS).

Enfin, pour pouvoir utiliser la fonction roundf() de C99, il faut faire l'édition de liens avec la bibliothèque mathématique en ajoutant -lm. (Voir aussi la variable LDLIBS dans la documentation sur make fournie la semaine passée.)

Notes :

  1. Le code fourni compile avec moult « warnings ». Nous ne vous demandons pas de corriger ces erreurs, mais simplement d'écrire un Makefile qui parvient à produire un exécutable main.

  2. Il n'est pas nécessaire de lancer le main en question. Si vous l'avez fait, vous pouvez simplement le quitter en tapant Ctrl-C.

  3. Pour avoir tous les points, votre Makefile devra non seulement produire un exécutable main lorsque l'on tape make, mais aussi avoir toutes les dépendances correctement spécifiées.

III. Utilisation d'une bibliothèque extérieure : FUSE

La bibliothèque FUSE (« système de fichiers dans l'espace utilisateur », « Filesystem in USErspace » en anglais) permet de créer des programmes C qui peuvent simuler un système de fichiers directement intégrable dans le système de fichiers Linux. Cela nous sera utile pour le projet en permettant par exemple de lire directement dans Linux les fichiers présents dans les disques artificiels que vous créerez dans votre projet (au travers d'une interface que vous écrirez en C ; n'est-ce pas magnifique ?).

Le but de cette partie est simplement, toujours dans le même esprit de ces trois semaines préparatoire, de s'assurer que vous avez le bon environnement et les bons outils pour mener à bien votre projet.

Principes de FUSE

FUSE est séparé en deux parties au niveau de son implémentation :

  • le module fuse du noyau (« fuse kernel module ») livré avec le noyau Linux ;
  • la librairie libfuse dans l'espace utilisateur, qui vous permet de dialoguer avec ce module et qu'il faudra donc utiliser.

Installation de libfuse pour les machines personnelles (autres que les VM de l'Ecole)

Si nécessaire (si vous utilisez votre propre distribution Linux, qui supporte les packages deb), vous pouvez simplement taper :

sudo apt-get install libfuse2 libfuse-dev pkg-config

qui se chargera d'installer ce qui est nécessaire à la bibliothèque FUSE.

Note: pkg-config est un outil que nous utiliserons dans le Makefile [fin de note]

Si vous n'utilisez pas Linux, il est impératif de télécharger et d'installer une machine virtuelle Linux comme précisé lors de la première semaine. FUSE venant en partie avec le noyau Linux, les différentes solutions proposées (pour OSX notamment) sont trop souvent incomplètes et vous feront sûrement perdre plus de temps. Il est impératif que votre implémentation fonctionne avec le système d'exploitation officiel du cours tel qu'il vous a été présenté la semaine passée.

Faire fonctionner un exemple

Pour cette étape, nous ne vous fournissons un exemple simple hello.c (dans provided/fuse. Celui-ci provient de la distribution officielle de libfuse en guise d'exemple basique pour utiliser cette bibliothèque.

Comme nous ne vous demandons aucune modification ici, vous pouvez travailler directement dans provided/fuse. Ouvrez avec un éditeur les fichiers fournis pour regarder globalement ce qu'ils contiennent. Essayez de comprendre la structure et les idées principales sans vous noyer dans les détails (que vous ne connaissez pas encore). Demandez de l'aide/des explications sur tel ou tel aspect de syntaxe de C si ce n'est pas clair (mais pas sur les détails de FUSE, ce n'est pas le but ici).

Commençons par tester si ce fichier compile. Nous fournissons pour cela un Makefile (dans provided/fuse). Il contient déjà les règles de compilation spécifiques pour compiler hello : allez voir et comprenez ce qui est écrit (et comment ça marche).

Il vous suffit donc de compiler simplement avec :

make

Vous devriez alors avoir un exécutable hello dans votre répertoire courant (done/).

Note : si vous avez activé certains warning de compilation, vous pouvez justement avoir des messages de warning (le code fourni n'est pas 100% propre ; comme quoi...). Ignorez les simplement. [fin de note]

Ce que fait FUSE, c'est de mounter un système de fichiers à un endroit (répertoire) de votre système de fichiers actuel ; on appelle cela un « point de montage » (ou « mount point »). Pour tester le programme hello, il vous faut alors un point de montage. Je vous propose pour cela de créer le répertoire hello-dir dans /tmp.

mkdir /tmp/hello-dir

et vérifiez (pour mieux comprendre la suite) qu'il ne contient effectivement rien :

ls -l /tmp/hello-dir

Vous pouvez alors le passer comme argument à hello :

./hello /tmp/hello-dir

À première vue, il ne s'est rien passé. Faites les deux commandes suivantes :

ps auxw | grep hello | grep -v grep
mount | grep hello

Si tout se passe normalement, la commande ps confirmera que le processus hello tourne toujours, et la commande mount que nous avons un système de fichiers de plus, géré par fuse.hello.

Allez voir ce qui s'y trouve :

ls -l /tmp/hello-dir
cat /tmp/hello-dir/demo_file.txt

(ça ne doit pas être vide).

Vous pouvez même copier ce fichier demo_file.txt, p.ex. : 

cp /tmp/hello-dir/demo_file.txt  ../../done/msg.txt
ls -l ../../done/
cat ../../done/msg.txt

Le fichier /tmp/hello-dir/demo_file.txt est bien un vrai fichier pour votre OS, sauf qu'il est « offert » par l'exécutable hello que vous venez (de compiler et) de lancer.

Une fois terminé, unmountez (ou « démontez ») ce système de fichiers à l'aide de la commande :

fusermount -u /tmp/hello-dir

Pour montrer que vous avez bien fait cette partie, ajoutez à votre dépôt git les fichiers (de done !) msg.txt (qui doit être dans done !!).

Avant de quitter cette série, assurez vous d'avoir bien compris ce que vous avez fait, ce que fait le programme hello (sans entrer dans les détails de FUSE, ni de concept de C avancés, bien sûr ! Simplement comprendre les principaux concepts) et pourquoi il est si puissant. En semaine 8, vous écrirez un programme équivalent pour pouvoir mounter le système de fichiers créé dans le projet.
